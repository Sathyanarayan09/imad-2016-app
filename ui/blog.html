How to become a self taught programmer?

by Sathyanarayan

Posted on October 24, 2016 at 9:00 PM

Previous Next

Building a learning environment

Any laptop or desktop is good to learn. However it cannot be a phone or a tablet (no programming on iphones folks!). It doesn't matter if it's Mac, PC or Linux as long as you have admin access and you are able to install anything you need. Learning programming is best in a quiet environment, with little to no interruptions. Set aside at the very minimum one hour per day, ideally a good three hours. Self-taught programming in 2016 is done with an relatively decent Internet connection. It means you can download a 50MB package without interruptions. Contrary to perceptions, programming is best with social feedback, so make sure that you know someone who can give input and challenge you on a regular basis. The stronger the feedback, the better! You can also follow programmer's blogs and read in a daily basis as part of your daily routine. Most don't showcase code and instead explain how they design software or solve critical blogs, which is essential for your culture. I read Coding Horror, Daily WTF, the Netflix blog, igvita, Facebook Engineering, the Twitter engineering blog and a few others.

Online resources

Currently, the best ressources are not books but online resources. Without any background in programming, you can start with the free resource Code Academy. It's important to follow each lesson and take your time. I love its interactive tools and relevant examples. You can follow the basic HMTL, CSS, javascript courses there. Your goal should be on building a simple static website in 2 or 3 weeks after your codeacademy course. I recommend you put your own online resume, with a few pictures, and links to your online presence. Pick yourself a code editor (take a look at Sublime Text), a free graphics program (Gimp or if you can Photoshop), and you have all the tools to ship your first site. Up next are either TeamTreeHouse, CodeSchool or CodeAvengers. They are more career-oriented, and none is necessarily better than the other. It's important however to choose one and stick to it. I would recommend at this point to pick one that has the best javascript / node.js course. Three months later, your goal should be on building a simple to-do list site after the first javascript course, and be able to deploy it on the Internet.
